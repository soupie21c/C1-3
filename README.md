# 자동화 도구 비교 분석 보고서

## 1. 프로젝트 개요

**구현한 워크플로우**: 업무 요청 접수 → 긴급 여부에 따라 Slack 알림 분기

- **Trigger**: Google Sheets "업무 요청 접수" 시트에 새 행이 추가되면
- **조건 분기**: "우선순위" 열 값이 "긴급"인지 "일반"인지에 따라 분기
- **Action A (긴급)**: Slack `긴급요청1` 채널에 요청자/내용 포함 메시지 전송
- **Action B (일반)**: Slack `일반요청1` 채널에 요청자/내용 포함 메시지 전송

동일한 구조를 **Zapier**와 **Make** 두 도구로 각각 구현하고 실행 결과를 확인했다.

---

## 2. 사용한 도구

| 도구 | 조건 분기 기능 이름 |
|---|---|
| Zapier | Paths by Zapier |
| Make | Router |

---

## 3. 구현 과정 요약

### Zapier
1. Google Sheets 앱 연결 → Trigger 이벤트 "New Spreadsheet Row" 선택
2. 1~2단계 사이에 "Paths by Zapier" 추가 → 자동으로 두 경로(Path) 생성
3. 각 경로에 "Path conditions" 설정 (우선순위 = 긴급 / 우선순위 = 일반)
4. 각 경로 하위에 Slack "Send Channel Message" 액션 추가, 채널 및 메시지 텍스트(요청자/내용 필드 삽입) 구성
5. Publish로 Zap 활성화 후 실제 데이터로 실행 확인

### Make
1. Google Sheets 모듈 "Watch New Rows" 추가 및 연결
2. 다음 모듈로 "Router" 추가 → 1st, 2nd 두 경로 생성
3. 각 경로 연결선에 필터(Filter) 설정 (우선순위 = 긴급 / 우선순위 = 일반)
4. 각 경로 끝에 Slack "Send a Message" 모듈 추가, 채널명(#긴급요청1, #일반요청1) 및 텍스트(요청자/내용 필드 삽입) 구성
5. Slack 채널에 Make 봇을 초대(channel_not_found 에러 해결)한 후 "Run once"로 실행 확인

## 4. 비교 항목

| 비교 항목 | Zapier | Make |
|---|---|---|
| UI/UX | 단계가 세로로 나열되는 선형(리스트) 구조. 각 단계를 순서대로 클릭하며 진행해 처음 사용자도 흐름을 놓치기 어려움 | 캔버스 위에 모듈이 노드-연결선 형태로 배치되는 시각적 구조. 전체 흐름을 한눈에 보기 좋지만 자유도가 높은 만큼 초반 방향을 잡기까지 시간이 걸림 |
| 설정 난이도 | 조건 분기(Paths) 설정 시 필드 선택→조건→값 입력의 단계가 명확해 따라 하기 쉬움 | 조건(Filter)을 모듈이 아니라 연결선 위 아이콘을 클릭해 들어가야 해 처음에는 위치를 찾기 어려웠음 |
| 연동 서비스 범위 | Google Sheets, Slack 등 주요 서비스 연동이 매끄러웠고 필드 매핑(COL A, COL B 등)이 직관적으로 표시됨 | 연동 자체는 문제없었으나, Slack 채널 지정 시 이름을 직접 입력(#채널명)해야 했고 봇을 채널에 별도로 초대해야 하는 추가 단계가 필요했음 |
| 무료 플랜 범위 | 무료 플랜에서도 Paths(조건 분기) 사용 가능, 다단계 워크플로우 제작에 큰 제약 없었음 | 무료 플랜에서 Router 사용 가능, 다만 기본 폴링(실행 확인) 주기가 15분으로 설정되어 즉시성은 떨어짐 |
| 실행 로그 확인 방식 | Zap history 메뉴에서 실행 이력 확인, 각 단계 테스트 시 "Data in/Data out" 탭으로 입출력 데이터를 바로 확인 가능 | 각 모듈에 초록 체크/빨간 에러 아이콘이 바로 표시되고, 클릭하면 "Input/Output/Error" 상세 로그가 함께 표시되어 에러 원인 파악이 쉬움 (예: channel_not_found 에러 메시지) |

---

## 5. 각 도구의 장단점

### Zapier
**장점**
- 단계별 진행 방식이라 초보자도 순서를 헷갈리지 않고 따라갈 수 있음
- 필드 매핑 UI(리스트 클릭 삽입)가 직관적

**단점**
- 복잡한 워크플로우가 될수록 전체 구조를 한눈에 파악하기 어려움 (세로로 길게 나열됨)

### Make
**장점**
- 캔버스 형태로 전체 워크플로우 구조를 한눈에 볼 수 있어 조건 분기가 많아져도 흐름 파악이 쉬움
- 에러 발생 시 원인(Code, Origin 등)을 구체적으로 보여줘 디버깅이 편리함

**단점**
- Slack 등 일부 연동에서 채널을 봇에 직접 초대해야 하는 등 추가 설정 단계가 있어 초기 진입장벽이 다소 있음
- 조건(Filter) 설정 위치가 직관적이지 않아 처음에는 찾기 어려웠음

---

## 6. 어떤 상황에 적합한지

- **Zapier**: 자동화를 처음 접하거나, 단순하고 명확한 단계별 흐름을 선호하는 경우에 적합
- **Make**: 여러 조건 분기가 얽힌 복잡한 워크플로우를 설계하거나, 전체 구조를 시각적으로 파악하며 작업하고 싶은 경우에 적합

## 7. 보안 관련 처리

본 보고서 및 첨부 캡처 자료에는 Slack 워크스페이스 계정 이메일, API 토큰 등 민감정보는 포함하지 않았으며, 필요한 경우 마스킹 처리하였다.

# 프로젝트 2: 자유 주제 자동화 설계 및 구현

## 1. 자동화할 반복 업무

매일 이메일로 들어오는 업무 요청을 확인하고, 스프레드시트에 기록한 뒤, 팀 채널에 공유하는 반복 업무를 자동화한다. 기존에는 이메일을 하나씩 열어 내용을 확인하고 수동으로 시트에 옮겨 적은 뒤, 다시 Slack에 요약해서 알리는 과정을 반복해야 했다.

## 2. 자동화 도구 선정 및 선정 이유

**선정 도구**: Make

**선정 이유**: 프로젝트 1(Zapier vs Make 비교)에서 두 도구를 직접 사용해본 결과, Make는 캔버스 형태로 전체 워크플로우 구조를 한눈에 볼 수 있어 Gmail → Router → (긴급/일반) → Sheets/Slack처럼 여러 단계와 두 개의 분기 경로가 함께 있는 복잡한 흐름을 설계하고 점검하기에 더 적합하다고 판단했다. 또한 에러 발생 시 원인(Error code, Origin 등)을 구체적으로 보여줘 디버깅이 수월했던 경험도 선정에 영향을 주었다.

## 3. 워크플로우 흐름 설명

```
Gmail (Watch emails)
  └─ 조건: 제목에 "[요청]" 포함된 새 메일 감지
        ↓
Router (조건 분기)
  ├─ 1st 경로 "긴급": 제목에 "[긴급]" 포함
  │     ├─ Google Sheets: "업무 요청 접수" 시트에 새 행 추가
  │     │     (요청자=발신자명, 내용=제목, 우선순위="긴급")
  │     └─ Slack: #긴급요청1 채널에 알림 메시지 전송
  │
  └─ 2nd 경로 "일반": 제목에 "[긴급]" 미포함
        ├─ Google Sheets: "업무 요청 접수" 시트에 새 행 추가
        │     (요청자=발신자명, 내용=제목, 우선순위="일반")
        └─ Slack: #일반요청1 채널에 알림 메시지 전송
```

**단계별 설명**

1. **Trigger**: Gmail에서 제목에 "[요청]"이 포함된 새 이메일이 도착하면 워크플로우가 시작된다.
2. **조건 분기**: Router 모듈이 이메일 제목에 "[긴급]"이라는 문자열이 포함되어 있는지를 기준으로 두 경로로 나눈다.
3. **Action (긴급 경로)**: 스프레드시트에 우선순위를 "긴급"으로 기록하고, `긴급요청1` Slack 채널에 발신자와 제목을 포함한 메시지를 보낸다.
4. **Action (일반 경로)**: 스프레드시트에 우선순위를 "일반"으로 기록하고, `일반요청1` Slack 채널에 동일한 형식의 메시지를 보낸다.

## 4. 요구사항 충족 확인

| 요구사항 | 충족 여부 |
|---|---|
| Trigger 1개 이상 | Gmail 새 메일 감지 |
| Action 2개 이상 | Google Sheets 기록 + Slack 메시지 전송 |
| 조건 분기 1개 이상 | Router (제목 내 "[긴급]" 포함 여부) |
| 각 분기 1회 이상 실행 확인 | 긴급 경로 1건, 일반 경로 2건 실제 실행 및 결과 확인 완료 |
| Trigger 발생 시 자동 실행 | Make 시나리오 활성화(스케줄 실행) 상태로 운영 |

## 5. 실행 결과

- Google Sheets "업무 요청 접수" 시트에 이메일 기반 데이터(요청자, 내용, 우선순위)가 자동으로 기록됨을 확인
- Slack `긴급요청1` 채널에 제목 "[요청][긴급] 서버 점검 필요" 메일에 대한 알림 메시지 수신 확인
- Slack `일반요청1` 채널에 제목 "[요청] 문서 검토 부탁드립니다" 메일에 대한 알림 메시지 수신 확인
